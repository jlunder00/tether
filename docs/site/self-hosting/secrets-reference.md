# Secrets Reference

## Auth config (`~/.tether-config/auth_config.yaml`)

Set via `make configure-auth` or edit the file directly.

| Key | Default | Notes |
|-----|---------|-------|
| `jwt.secret` | _(required)_ | Auto-generated by `make install`. |
| `cookie.secure` | `true` | Set `false` for local HTTP access (not localhost). |
| `cors.allowed_origins` | `http://localhost:5173,http://localhost:8000` | Comma-separated list. Add your domain if accessing remotely. |

### GitHub OAuth

Set via `make configure-github`. All keys are optional — leave `enabled: false` to skip.

| Key | Notes |
|-----|-------|
| `oauth.github.enabled` | Set `true` to show the GitHub login button. |
| `oauth.github.client_id` | |
| `oauth.github.client_secret` | |
| `oauth.github.callback_url` | Must match the callback URL in your GitHub OAuth app settings. |

### Google OAuth

Set via `make configure-google`. All keys are optional — leave `enabled: false` to skip.

| Key | Notes |
|-----|-------|
| `oauth.google.enabled` | Set `true` to show the Google login button. |
| `oauth.google.client_id` | |
| `oauth.google.client_secret` | |
| `oauth.google.callback_url` | Must match your Google Cloud Console OAuth configuration. |
| `oauth.google.integration_callback_url` | Required for Google Calendar integration. |

## App config (`~/.tether-config/app_config.yaml`)

Edit directly or use `make configure-*`.

| Key | Default | Notes |
|-----|---------|-------|
| `telegram.bot_token` | _(none)_ | Optional. Set via `make configure-telegram`. Get from [@BotFather](https://t.me/BotFather). |
| `telegram.chat_id` | _(none)_ | Optional. The chat ID the Telegram bot responds to. |
| `models.*` | See [Configuration](./configuration#app-configyaml) | Model for each pipeline role. |
| `pipeline.*` | See [Configuration](./configuration#app-configyaml) | Planning loop tuning. |
| `server.api_port` | `8000` | |
| `server.mcp_port` | `5001` | |

## LLM credentials (environment variables)

At least one backend is required. The active backend is set in the web dashboard under Settings → LLM.

| Variable | Backend |
|----------|---------|
| `ANTHROPIC_API_KEY` | Anthropic API |
| `OPENAI_API_KEY` | OpenAI or any OpenAI-compatible endpoint (set `OPENAI_BASE_URL` to use a custom endpoint) |
| `OPENROUTER_API_KEY` | OpenRouter |
| AWS credentials | AWS Bedrock (uses ambient boto3 credentials) |
| _(Claude Code logged in)_ | Claude Code CLI — always available as a fallback if `claude` is installed |

---

## Database passwords (`~/tether/.env`)

Set via `make configure-db`.

| Variable | Default |
|----------|---------|
| `POSTGRES_PASSWORD` | `tether_dev` |
| `TETHER_APP_PASSWORD` | `tether_app_dev` — must match the password used when creating the `tether_app` role |

::: danger Change these defaults
The default passwords are public. Change them before exposing Tether to a network.
:::

## Environment variables (advanced)

| Variable | Default | Notes |
|----------|---------|-------|
| `TETHER_CONFIG_DIR` | `~/.tether-config` | Where YAML config files are read from. |
| `TETHER_COMPOSE_DIR` | `~/tether` | Where `make configure-db` writes the Docker Compose `.env`. |
| `TETHER_USER_ID` | _(empty)_ | Scopes MCP queries to a specific user. Set if using MCP in a multi-user setup. |
